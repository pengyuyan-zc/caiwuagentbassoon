# 开票主体税务凭证生成 — 详细规则

## 凭证分录结构

每张发票生成 1 张凭证，顺序：借 → 贷 → 贷 → 贷

| 方向 | 科目编码 | 科目名称 | 金额来源 | 客户字段 |
|------|---------|---------|---------|---------|
| 借 | 1122 | 应收账款 | 开票总额 | 填（客户编码+名称） |
| 贷 | 6001.xx.xx | 主营业务收入 | 不含税金额 | 不填 |
| 贷 | 2221.01.02 | 销项税额 | 增值税金额（≠0 时） | 不填 |
| 贷 | 6001.xx.xx | 主营业务收入 | 差额开票金额（>0 时） | 不填 |

**差额开票逻辑：**
- 差额开票金额 = 0：贷方 2 条（6001 + 2221）
- 差额开票金额 > 0：贷方 3 条（6001不含税 → 2221税 → 6001差额），顺序固定

## 校验公式

`开票总额 = 不含税金额 + 差额开票金额 + 增值税金额`

## 业务类型 → 科目映射

| 内部类型分类 | 主营业务收入科目 |
|------------|--------------|
| 派遣业务 | 6001.02.00 派遣收入 |
| 代理业务 | 6001.03.01 代理收入 |
| 承揽业务 / 小时间承揽 | 6001.04.01 承揽收入 |
| 其他未识别 | 6001.04.01 承揽收入（默认） |

## 摘要生成规则

`摘要 = 客户简称 + YYYYMM + 月开票`

客户简称取自客户明细表全称；客户明细未命中时，用发票原始名称去掉地区前缀。

## 客户明细匹配规则

从 `客户明细2026*.xlsx` 加载，按「客户名称」列匹配：
1. 精确匹配
2. 模糊匹配（含公司名关键词，去掉「有限公司」「股份有限公司」等后缀）
3. 均未命中 → `cust_code=None`，1122 借方行客户字段留空

## 错误处理

- `开票总额 <= 0` 或 Excel 单元格含 `#VALUE!` → 自动跳过该行
- 客户明细未找到 → 警告打印未匹配客户名，继续生成
- 科目表/模板未找到 → 返回 `success: False`，带错误消息

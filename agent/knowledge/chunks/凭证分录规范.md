# 凭证分录规范

## 开票凭证生成规则

### 全额开票（3条分录）

当增值税金额 > 0 时，使用全额开票规则：

```
借 1122 应收账款        = 开票总额（填客户编码/名称）
贷 6001.XX.XX 主营业务收入 = 不含税金额
贷 2221.01.02 销项税额   = 增值税金额
```

### 差额开票（4条分录）

当开票类型包含"差额"关键词时：

```
借 1122 应收账款        = 开票总额
贷 6001.XX.XX 主营业务收入 = 不含税金额
贷 2221.01.02 销项税额   = 增值税金额（≠0 时才写）
贷 6001.XX.XX 主营业务收入 = 差额开票金额（>0 时才写）
```

### 无税额仅差额（2条分录）

当增值税金额 = 0 且存在差额时：

```
借 1122 应收账款        = 开票总额
贷 6001.XX.XX 主营业务收入 = 差额开票金额
```

## 摘要生成规则

摘要格式：`客户简称 + YYYYMM + "月开票"`

示例：
- 客户"深圳市腾讯计算机系统有限公司"，简称"腾讯"，2024年6月 → "腾讯202406月开票"
- 客户简称从客户明细表匹配，未命中则去掉发票名称的地区前缀

## 客户简称匹配规则

1. 优先从「客户明细表」中精确匹配客户名称
2. 未匹配时，去掉发票名称的地区前缀（如"深圳市"、"广东省"等）
3. 仍无法匹配时，使用发票全称并在摘要中注明

## 金额处理规则

- 开票总额 = 不含税金额 + 增值税金额
- 不含税金额 = 开票总额 / (1 + 税率)
- 税率从发票数据中读取（如 6%、9%、13%）
- 所有金额保留 2 位小数

## 数据源要求

开票主体 Excel 必须包含以下列：
- 审批编号（唯一标识）
- 客户/项目名称
- 开票总额
- 不含税金额
- 增值税金额
- 开票类型（判断全额/差额）
- 业务类型（判断收入科目子目）
